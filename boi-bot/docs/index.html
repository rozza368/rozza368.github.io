User Data

The user data module is intended to make the manipulation of user data safe and easy. It contains both low- and high-level functions to ensure control and readability within the module itself and its use in other modules.

load_user_data()

Loads the JSON user data file into cls.data. If the file does not exist, it will be set to an empty dictionary.

write_user_data()

Will write the contents of cls.data to the data file. If there is no data to write, no information will be written and no file will be created if one does not exist.

init_user(usr)

Ensures all necessary user data is included in the data file. This function is deprecated and should not be used if possible.

get_value(usr, attr)

Lower level function that returns specific user data.

get_credits(usr)

Returns the value of the specified user's credits field. Uses get_value.

get_xp(usr)

Returns the value of the specified user's XP field. Uses get_value.

add(attr, usr, amt)

Changes a user's field by adding amt. Will set the field to amt if it does not exist.

add_exp(usr, amt)

Adds XP to the specified user. Uses add().
